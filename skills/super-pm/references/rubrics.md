# Product Management Interview Evaluation Rubrics

This document defines the evaluation criteria for the Product Management Interviewer. Feedback is provided across four core pillars: Product Vision, Strategic Thinking, Analytical Rigor, and Communication.

## 1. Product Vision
*Focus: Clarity of "Why," identifying the core user problem, and user empathy.*

| Level | Description | Key Markers |
| :--- | :--- | :--- |
| **Excellent** | Exceptional ability to identify the "pain behind the pain." Connects user needs deeply to a compelling long-term vision. | - Identifies non-obvious user pain points.<br>- Clearly articulates a "North Star" mission.<br>- Prioritizes features based on high user impact and vision alignment. |
| **Good** | Clearly identifies the target user and their primary problem. Vision is logical but perhaps lacks a unique "wow" factor. | - Defines specific user segments.<br>- Articulates a clear problem statement.<br>- Proposes solutions that directly address the identified problem. |
| **Needs Improvement** | Vague or generic problem statement. Jumps to solutions without establishing a clear "Why." | - Solution-first thinking.<br>- Fails to define a specific target user.<br>- Vision is missing or contradicts the proposed solution. |

## 2. Strategic Thinking
*Focus: Business viability, competitive landscape, and ecosystem impact.*

| Level | Description | Key Markers |
| :--- | :--- | :--- |
| **Excellent** | Anticipates market shifts and competitor moves. Understands complex ecosystem dynamics and identifies unique leverage points. | - Considers second-order effects of decisions.<br>- Identifies sustainable competitive advantages (moats).<br>- Balances short-term wins with long-term strategic goals. |
| **Good** | Sound understanding of the business model and competitive landscape. Considers basic trade-offs and ecosystem players. | - Identifies primary competitors.<br>- Understands how the product makes money or creates value.<br>- Evaluates obvious trade-offs (e.g., speed vs. quality). |
| **Needs Improvement** | Narrow focus on features without considering the broader business context. Ignores competition or market realities. | - Fails to consider how the product fits into the market.<br>- Ignores business constraints or viability.<br>- No mention of trade-offs or ecosystem impact. |

## 3. Analytical Rigor
*Focus: Use of data, metrics (North Star, Counter-metrics), and logical structuring.*

| Level | Description | Key Markers |
| :--- | :--- | :--- |
| **Excellent** | Develops a sophisticated metrics framework. Uses data-driven logic to navigate ambiguity and defend decisions. | - Defines a clear North Star metric and relevant counter-metrics.<br>- Breaks down complex problems into MECE (Mutually Exclusive, Collectively Exhaustive) components.<br>- Uses "back-of-the-envelope" estimations to validate assumptions. |
| **Good** | Proposes logical metrics and uses a structured approach to problem-solving. Shows basic data literacy. | - Identifies key success metrics.<br>- Follows a clear framework (e.g., CIRCLES, 5 Whys).<br>- Provides logical reasoning for prioritization. |
| **Needs Improvement** | Illogical or missing metrics. Poorly structured response that is difficult to follow. | - Guesses at metrics without justification.<br>- Disorganized "stream of consciousness" delivery.<br>- Fails to use data or logic to support claims. |

## 4. Communication
*Focus: Executive presence, structured delivery, and handling of pushback.*

| Level | Description | Key Markers |
| :--- | :--- | :--- |
| **Excellent** | Highly engaging and persuasive. Handles tough pushback with grace and adapts delivery to the audience (executive vs. technical). | - Compelling storytelling and presence.<br>- Proactively addresses potential concerns.<br>- Synthesizes complex ideas into simple, actionable insights. |
| **Good** | Clear, concise, and well-structured. Maintains professional poise and responds effectively to questions. | - Uses signposting (e.g., "I have three points...").<br>- Listens actively and clarifies expectations.<br>- Logical flow of ideas. |
| **Needs Improvement** | Rambling, repetitive, or defensive. Lacks structure and fails to engage the interviewer. | - Difficult to follow or "lost in the weeds."<br>- Becomes defensive or flustered under pressure.<br>- Fails to summarize or conclude effectively. |
